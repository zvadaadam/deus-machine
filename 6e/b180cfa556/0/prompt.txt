### **#1 Shutdown exits immediately without waiting for children to drain**
In `start.ts`, the `shutdown()` function sends `SIGTERM` to child processes then immediately calls `process.exit(0)`. The agent-server has graceful shutdown logic with 30s turn draining, but gets zero time to execute it. This can orphan processes, leave ports occupied on restart, and `clearServerInfo()` runs before children actually exit — causing `deus status` to report "not running" while processes are still alive.
File: `apps/cli/src/start.ts:90-103`
Fix: await child `exit` events (with a reasonable timeout) before calling `process.exit`.
---
### **#2 Memory leak: stdout listener never removed after pattern match**
In `startProcess()`, the `child.stdout.on("data", ...)` listener appends to a `buffer` string indefinitely. After the `waitFor` regex matches and the promise resolves, the listener is never removed. For long-running processes (backend can run for days), the buffer grows without bound. Practical growth is slow (backend produces a few KB/hour of stdout), but it's an unbounded accumulation with no cleanup.
File: `apps/cli/src/start.ts:312-329`
Fix: remove the `data` listener (or null out the buffer) after the regex matches.
