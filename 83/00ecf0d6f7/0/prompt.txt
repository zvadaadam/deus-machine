### **#1 Model selection from welcome screen is silently dropped**
`WelcomeView` lets the user pick a model and passes it via `onSendMessage(repoId, content, model, branch)`. In `MainLayout.tsx`, `handleSendFromWelcome` receives the `model` parameter but never stores it in `pendingWelcomeMessageRef` (which only has `{ message, workspaceId }`). **File:** `apps/web/src/app/layouts/MainLayout.tsx:177-216`
**Type:** Bug
---
### **#2 `/repos/clone` endpoint has no path sanitization**
The new `POST /api/repos/clone` endpoint accepts `targetPath` from the request body and passes it directly to `fs.mkdirSync(parentDir, { recursive: true })` and `spawn("git", ["clone", ..., targetPath])` with zero validation. Compare to `POST /repos` which uses `fs.realpathSync()`, `fs.accessSync()`, and validates it's a git root. This endpoint is reachable via the cloud relay (`delegateToRoute` in `commands.ts`), so an authenticated remote user could clone to arbitrary filesystem locations.
**File:** `apps/backend/src/routes/repos.ts:124-185`
**Type:** Security
---
### **#3 `useRepoBranches` fires unconditionally in `BranchSelector`**
The old code gated the branch query on the popover being open: `useBranches(open ? workspacePath : null)`. The new code calls `useRepoBranches(repoId)` unconditionally on mount, and the hook in `workspace.queries.ts` only gates on `!!repoId` — no `open` condition. Every `BranchSelector` instance fires a WS request immediately, even if the user never opens the dropdown.
**CLAUDE.md rule:** *"Conditional polling — always gate polling on relevant state... don't poll idle workspaces, don't poll settings, don't poll repos."*
**File:** `apps/web/src/features/workspace/ui/BranchSelector.tsx:27`
**Type:** CLAUDE.md violation
