### #1 Security: Git argument injection via unsanitized `source_branch`

`source_branch` is validated only as `z.string().optional()` and flows directly into `git fetch origin <branch>` at line 197 with no `--` end-of-options separator. A crafted value like `--upload-pack=/bin/malicious` would be interpreted by git as an option flag, enabling arbitrary command execution. Combined with the app's open CORS policy, a malicious website could exploit this via `POST http://localhost:{port}/api/workspaces`.

**Fix:** Add branch name validation in `schemas.ts` and/or insert `"--"` before ref arguments:

```ts
// schemas.ts
source_branch: z.string()
  .refine((s) => !s.startsWith("-"), "Branch name must not start with a dash")
  .optional(),

// workspaces.ts line 197
await execFileAsync("git", ["fetch", "origin", "--", parent_branch], { ... });
```

Apply the same to `target_branch`.

**File:** `apps/backend/src/lib/schemas.ts:89`, `apps/backend/src/routes/workspaces.ts:197`

### #3 Fragile `cursor-grab` specificity on RepositoryItem

`SidebarRow` at line 61 applies `cursor-grab [&_*]:cursor-grab`, which targets all descendants. The action buttons inside (lines ~93, ~110) apply `cursor-pointer [&_*]:cursor-pointer`. Both selectors have identical specificity `(0,1,0)`, so the winner depends on Tailwind's CSS generation order — non-deterministic from the developer's perspective. This means action buttons may show a `grab` cursor instead of `pointer`.

The drag cursor previously lived in `DraggableRepository.tsx` (the drag host), which is the correct location. Moving it into the inner presentation component splits responsibility and creates this conflict.

**File:** `apps/web/src/features/sidebar/ui/RepositoryItem.tsx:61`

### #4 N+1 git subprocess loop in `backfillGitOriginUrls`

The startup backfill spawns one sequential `git remote get-url origin` subprocess per repo with `git_origin_url IS NULL` — no concurrency cap, no batching.

> CLAUDE.md: "Cap concurrent git subprocesses and queue excess work to prevent CPU spikes and I/O contention." and "Avoid per-item hooks that spawn git processes."

Severity is bounded (one-time backfill, not steady-state), but with many repos the sequential loop delays startup and the `invalidate(["workspaces"])` call. Use `Promise.allSettled` with a concurrency limit (e.g., 5).

**File:** `apps/backend/src/server.ts:50-65`
