### **#1 Legacy database path will silently lose existing users' data**
The legacy database fallback in `backend-process.ts` was blindly renamed from `com.opendevs.ide/opendevs.db` to `com.deus.ide/deus.db`. The entire purpose of this legacy path is to find databases from a **previous** version. No user has ever had a database at `com.deus.ide/deus.db`.
Additionally, since `electron-builder.yml` changed `appId` to `com.deus.app`, Electron's `app.getPath("userData")` now returns `~/Library/Application Support/Deus/` instead of `~/Library/Application Support/OpenDevs/`. The `electronDbPath` also won't find existing databases.
**Result:** Existing users upgrading will silently start with an empty database — all repositories, workspaces, sessions, and messages lost.
**Fix:** The legacy path must keep the old location:
```ts
const legacyDbPath = join(app.getPath("appData"), "com.opendevs.ide", "opendevs.db");
```
**File:** `apps/desktop/main/backend-process.ts` (line ~41-45)
