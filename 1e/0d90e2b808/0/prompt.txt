## Review: Better Notifications
**Changes summary:** Two files modified to improve OS notification quality — notifications now show `"repo-name · workspace-title"` instead of truncated session IDs, and clicking any notification navigates to the relevant workspace/session.
### Issue Found
### **#1 Setup notification `onClick` missing `showWindow()` — window won't restore from minimized**
Session notifications use `navigateToWorkspace()` which calls `showWindow()` (Electron IPC → `show_main_window`) to restore minimized/hidden windows before navigating:
```ts
// navigateToWorkspace (line 54-64) — used by all session notifications
function navigateToWorkspace(workspaceId: string, sessionId: string): void {
  showWindow();  // ← Electron IPC to restore window
  useWorkspaceStore.getState().selectWorkspace(workspaceId);
  // ...
}
```
But the **setup failed** and **setup complete** notification `onClick` handlers skip this:
```ts
// Line 261 — setup failed
onClick: () => useWorkspaceStore.getState().selectWorkspace(ws.id),
// Line 271 — setup complete  
onClick: () => useWorkspaceStore.getState().selectWorkspace(ws.id),
```
The `window.focus()` call in `notificationService.ts` is a renderer-side call that doesn't invoke Electron's `show_main_window` IPC. On macOS, clicking a setup notification when the window is minimized/hidden will update internal state but the window won't actually come to the foreground.
**Fix:** Add `showWindow()` to both setup notification `onClick` handlers:
```ts
onClick: () => {
  showWindow();
  useWorkspaceStore.getState().selectWorkspace(ws.id);
},
```
**File:** `apps/web/src/features/session/hooks/useGlobalSessionNotifications.ts`, lines 261 and 271
---
**Overall:** Clean, well-structured change. The `FinishedEntry` interface, `formatBody` helper, and `navigateToWorkspace` utility are good abstractions. The notification bodies are now genuinely useful. Only the one inconsistency above needs fixing.
