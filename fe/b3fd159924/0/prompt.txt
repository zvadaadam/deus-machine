### **#1 `inflightPromise` never cleared on rejection — permanently breaks "Open in" feature**
If `loadInstalledApps()` throws an unhandled exception (e.g., `import("fs/promises")` fails, or any error outside the per-app try/catch), `inflightPromise` remains set to the rejected promise forever. The cleanup at line 459-460 (`cachedInstalledApps = installed; inflightPromise = null;`) only runs on the success path. Every subsequent call to `getInstalledAppsList()` hits `if (inflightPromise) return inflightPromise;` and immediately returns the stale rejected promise.
**Fix:** Use `try/finally` or wrap with `.finally(() => { inflightPromise = null; })`:
```ts
async function getInstalledAppsList(): Promise<InstalledApp[]> {
  if (cachedInstalledApps) return cachedInstalledApps;
  if (inflightPromise) return inflightPromise;
  if (process.platform !== "darwin") return [];
  inflightPromise = loadInstalledApps().finally(() => {
    inflightPromise = null;
  });
  return inflightPromise;
}
```
